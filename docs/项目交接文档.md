# 云师道 EduAgent 引擎后端项目交接文档

## 1. 项目概况

本项目是“云师道 EduAgent 引擎”的后端服务，面向乡村数学教师场景，提供登录鉴权、教案生成、课后反思、学情诊断、县域教研资源、课题推荐、专家预约、成长档案、报告导出、文件上传、语音转写等接口。

接口文档通过 Springdoc OpenAPI 自动生成，可直接导入 Apifox。

## 2. 技术栈

- JDK：17
- 后端框架：Spring Boot 3.5.12
- 构建工具：Maven
- 数据库：MySQL 8.x 或兼容版本
- ORM：MyBatis-Plus
- 鉴权：Spring Security + JWT
- 接口文档：Springdoc OpenAPI / Swagger
- AI 服务：阿里云 DashScope 千问系列模型
- 文件存储：本地 uploads 目录；生产环境可配置阿里云 OSS

## 3. 交付内容

建议发送给接收方以下内容：

```text
training-backend/
├── src/                         # 后端源码
├── pom.xml                      # Maven 依赖
├── sql/init.sql                 # 数据库初始化 SQL
├── sql/training_backend_database.sql  # 可直接发送的数据库初始化 SQL
├── docs/项目交接文档.md          # 本交接文档
├── README.md                    # 项目说明和接口示例
├── .env.production.example      # 生产环境变量模板
└── uploads/                     # 可选，已有上传文件/导出文件需要保留时一起发送
```

如果只是让别人本地跑通项目，必须提供代码、数据库 SQL、数据库连接信息、DashScope API Key。

如果要部署到服务器，还要提供生产环境变量、JWT 密钥、OSS 配置、服务器端口和域名信息。

## 4. 本地运行环境准备

接收方需要先安装：

- JDK 17
- Maven
- MySQL 8.x
- Git，可选

确认 Java 和 Maven：

```bash
java -version
mvn -version
```

## 5. 数据库初始化

数据库文件：

```text
sql/training_backend_database.sql
```

或使用原始文件：

```text
sql/init.sql
```

执行方式：

```bash
mysql -u root -p < sql/training_backend_database.sql
```

PowerShell 可使用：

```powershell
Get-Content .\sql\training_backend_database.sql | mysql -u root -p
```

默认数据库名：

```text
training_backend
```

SQL 文件会自动创建数据库和业务表。当前 SQL 包含 21 张表，覆盖用户、教案、反思、诊断、资源、课题、专家、成长档案、文件、通知、课程、课堂资源包等模块。

## 6. 修改数据库连接

本地开发配置文件：

```text
src/main/resources/application.yml
```

默认配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/training_backend?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
    username: root
    password: 123456
```

如果接收方本地 MySQL 账号密码不同，需要修改 `username` 和 `password`。

## 7. 必须配置的环境变量

本地开发至少需要配置千问 API Key：

```powershell
$env:DASHSCOPE_API_KEY="替换为真实 DashScope API Key"
```

Linux/macOS：

```bash
export DASHSCOPE_API_KEY="替换为真实 DashScope API Key"
```

不配置该 Key 时，普通登录、资源、课程等接口可以运行，但以下 AI 能力会失败：

- AI 生成教案
- 错题诊断
- 图片 OCR 诊断
- 语音转写
- AI 推荐课题

## 8. 启动项目

在项目根目录执行：

```bash
mvn spring-boot:run
```

默认端口：

```text
8080
```

健康检查：

```text
GET http://localhost:8080/api/health
```

Swagger 页面：

```text
http://localhost:8080/swagger-ui.html
```

OpenAPI JSON：

```text
http://localhost:8080/v3/api-docs
```

Apifox 可直接导入 `/v3/api-docs`。

## 9. 登录鉴权说明

无需登录的接口：

```text
POST /api/auth/register
POST /api/auth/login
GET  /api/health
GET  /swagger-ui.html
GET  /v3/api-docs
```

其他接口都需要请求头：

```text
Authorization: Bearer <登录接口返回的 token>
```

统一响应格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

常见业务码：

```text
200 成功
400 请求参数错误
401 未登录或 Token 无效
403 无权限
500 服务器内部错误
```

## 12. 文件上传说明

本地文件默认保存到：

```text
uploads/
```

本地访问前缀：

```text
/uploads
```

如果数据库里已有文件路径，并且需要保留旧的上传文件或导出报告，需要把 `uploads/` 目录也一起发给接收方。

如果只是让对方重新跑功能，可以不发旧的 `uploads/` 内容，项目会自动创建或使用新的上传目录。

## 14. 当前已知边界

当前项目已经有“课堂辅助资源包”接口，但没有真正封装独立的数字人后端服务。

现有课堂辅助接口只支持：

```text
GET  /api/classroom/assets
POST /api/classroom/assets/{assetId}/use
```

它返回的是 H5/WebGL 演示资源、离线包地址和使用记录，不包含数字人讲解、实时问答、动作控制、音视频流生成等能力。

如果后续要做真正数字人，建议新增：

```text
POST /api/digital-human/sessions
POST /api/digital-human/sessions/{id}/messages
POST /api/digital-human/explain
GET  /api/digital-human/assets
POST /api/digital-human/actions
```

## 15. 接收方启动检查清单

交接后请对方按顺序检查：

```text
1. JDK 17 是否安装
2. Maven 是否安装
3. MySQL 是否启动
4. 是否执行 sql/training_backend_database.sql
5. application.yml 数据库账号密码是否正确
6. 是否配置 DASHSCOPE_API_KEY
7. mvn spring-boot:run 是否启动成功
8. http://localhost:8080/api/health 是否返回 success
9. Apifox 是否能导入 http://localhost:8080/v3/api-docs
10. 注册、登录、携带 token 调用 /api/auth/me 是否成功
```
